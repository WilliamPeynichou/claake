# V0.03 — Marketplace complète + Sécurité (2026-03-16)

## Résumé

Claake passe d'un MVP vitrine à une **marketplace complète** : favoris, collections, paiements Stripe, reviews, recherche avancée, profils créateurs, notifications email, activity logs admin, et sécurisation production. Architecture desktop Tauri documentée.

---

## F1 — Favoris + Collections

### Backend (`src/modules/favorites/`)
- **Architecture complète** — Domain (FavoriteEntity, CollectionEntity), ports (FAVORITE_REPOSITORY, COLLECTION_REPOSITORY), Prisma repositories, mappers, transformers

- **FavoriteController** :

### Frontend

### Shared

## F5 — Activity Logs admin

### Prisma — nouveau modèle
```prisma
model ActivityLog {
  id         String   @id @default(uuid())
  actorId    String   @map("actor_id")
  actorEmail String   @map("actor_email")
  action     String
  targetType String   @map("target_type")
  targetId   String   @map("target_id")
  metadata   Json?
  createdAt  DateTime @default(now()) @map("created_at")
  @@index([action])
  @@index([targetType, targetId])
  @@index([createdAt])
  @@map("activity_logs")
}
```

## F6 — Reviews & Ratings

### Prisma
- Ajout `verifiedInteraction Boolean @default(false) @map("verified_interaction")` au model Review

### Backend (`src/modules/reviews/`)
- **ReviewEntity** — méthode `isOwnedBy(userId)`
- **CreateReviewUseCase** — logique de vérification :
  1. Pas de review existante (unique userId+agentId)
  2. Agent payant → vérifie Purchase → `verifiedPurchase = true`
  3. Agent gratuit → vérifie ChatSession → `verifiedInteraction = true`
  4. Aucun des deux → 403 "Vous devez utiliser cet agent avant de le noter"
  5. Crée review + recalcule `agent.rating` et `agent.reviewCount`
- **UpdateReviewUseCase** — vérification ownership + recalcul stats
- **DeleteReviewUseCase** — ownership OU admin `canManageReviews` + recalcul
- **ListReviewsUseCase** — par agentId, paginé, avec userName
- **ReviewController** :
  - `GET /v1/agents/:agentId/reviews` — public, paginé
  - `POST /v1/agents/:agentId/reviews` — auth
  - `PATCH /v1/agents/:agentId/reviews/:reviewId` — auth, owner
  - `DELETE /v1/agents/:agentId/reviews/:reviewId` — auth, owner ou admin
- **PrismaReviewRepository** — `computeAgentStats()` avec aggregate avg/count
- **ChatModule** exporte désormais `CHAT_SESSION_REPOSITORY`

### Frontend
- **ReviewForm** (`components/reviews/review-form.tsx`) — sélecteur étoiles + textarea
- **ReviewList** (`components/reviews/review-list.tsx`) — liste paginée avec badges "Achat vérifié" / "Interaction vérifiée"
- **Section reviews** sur la page agent détail

## F8 — Sécurisation

### Helmet — headers HTTP (`main.ts`)
- Content-Security-Policy, X-Frame-Options, HSTS, X-Content-Type-Options, Referrer-Policy
- `crossOriginEmbedderPolicy: false` pour compatibilité

### CORS production
- Prod : uniquement `process.env.WEB_URL`
- Dev : origins localhost (3000, 3001, 1420) + tauri://localhost

### Sanitization input
- **SanitizeInterceptor** (`common/interceptors/sanitize.interceptor.ts`) — strip récursif de tout HTML dans les body requests via `sanitize-html`
- Appliqué globalement sur toutes les routes

### Rate limiting
- ThrottlerModule global : 100 req/min par défaut
- `@SkipThrottle()` sur webhook Stripe
- Extensible avec `@Throttle()` granulaire par endpoint

### Dépendances ajoutées
- `helmet`
- `sanitize-html` + `@types/sanitize-html`
