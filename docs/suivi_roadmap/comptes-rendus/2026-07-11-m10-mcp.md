## Résultat

M10 livre une intégration MCP V1 sécurisée : les créateurs connectent des serveurs MCP HTTP,
découvrent et sélectionnent leurs outils, puis les soumettent à une revue admin indépendante.
Seuls les snapshots d'outils sélectionnés, approuvés et actifs sont exposés à la boucle tool
calling M8.

```txt
serveur MCP configuré
→ credentials chiffrés
→ initialize + tools/list
→ sélection outils
→ PENDING
→ revue admin APPROVED
→ ToolRegistry.prepare()
→ tools/call MCP pendant chat
```

## Backend

### Persistance et review

- Migration `0017_add_mcp` : `mcp_servers`, `mcp_tools`, statuts `DRAFT/PENDING/APPROVED/REJECTED/SUSPENDED`.
- Serveurs liés à un agent, suppression cascade ; outils uniques par serveur.
- Sous-CRUD owner/admin : création, édition, suppression, découverte, sélection et soumission.
- Revue admin dédiée : file `GET /admin/mcp/pending`, approbation, rejet motivé ou suspension.
- Toute découverte, sélection ou modification URL/auth ramène le serveur en `DRAFT`, désactive son usage et invalide l'approbation.
- Credentials write-only, chiffrés via service AES-256-GCM versionné existant ; jamais renvoyés par API.

### Transport MCP

- Client Streamable HTTP JSON-RPC : `initialize`, `notifications/initialized`, `tools/list` paginé et `tools/call`.
- Accepte réponses JSON et SSE MCP.
- SSRF guard appliqué avant chaque requête, redirections bloquées, HTTPS imposé en production.
- Headers credentials validés : limites, noms sûrs et headers réservés interdits.
- Timeout 10 s (plafond 30 s), réponse plafonnée à 2 Mo, 100 outils, schéma 64 Ko, arguments 256 Ko.
- Les erreurs de transport n'incluent ni credentials ni arguments.

### Chat / ToolRegistry

- `ToolRegistryService.prepare()` construit une fois par message un catalogue immuable d'outils intégrés et MCP.
- Port `MCP_TOOL_PORT` exporté de `McpModule` et injecté dans `ChatModule`, sans dépendance inverse.
- Aucune découverte MCP réseau pendant une conversation : snapshot DB exclusivement.
- Alias stable `mcp_<server>_<tool>` évite collisions providers/outils intégrés.
- Avant `tools/call`, statut, activation et sélection sont relus : rejet/suspension prend effet immédiatement.
- Quota M8 commun conservé : appel au-delà de cinq tentatives refusé avant exécution, y compris MCP.
- Résultat distant borné à 20 000 caractères.

## Shared et web

- `shared/types/mcp.ts` sépare contrats MCP privés des contrats chat publics.
- `apiClient.agents.mcp` : CRUD, découverte, sélection, soumission.
- `apiClient.admin.mcp` : file et décision de revue.
- `McpManager` dans édition agent : URL, auth write-only, découverte, sélection et soumission.
- Page `/admin/mcp` et lien de navigation admin pour revue MCP.
- `AgentChatConfig` ne contient ni URL MCP, ni credentials, ni états de revue.

## Limites V1

- Streamable HTTP prioritaire ; compatibilité SSE limitée aux réponses SSE, pas de transport SSE historique persistant.
- L'IP DNS est revalidée avant chaque requête ; épinglage socket strict contre DNS rebinding reporté à une couche HTTP bas niveau.
- Quota MCP dédié, concurrence et circuit breaker reportés ; plafond commun M8 par message actif.
- Test chat admin MCP via `/chat/:agentId?test=1` réutilise le chemin chat existant ; e2e serveur MCP live à ajouter avec environnement staging.
