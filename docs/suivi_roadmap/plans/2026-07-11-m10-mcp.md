# Plan — Milestone 10 MCP

Date : 2026-07-11
Branche : `main`
Réf : `docs/roadmap-claake-agents-chat.md` — Milestone 10 / Feature 8.3

## Objectif

Connecter un agent Claake à des serveurs MCP HTTP approuvés :

```txt
créateur configure serveur MCP
→ backend chiffre credentials
→ découverte tools/list bornée
→ snapshot tools sélectionnés
→ soumission review admin
→ approbation
→ ToolRegistry prépare catalogue intégré + MCP
→ tools/call pendant boucle provider M8
```

## Architecture retenue

- Nouveau module backend `mcp` structuré domain/application/infrastructure.
- Prisma source de vérité pour serveurs, outils découverts et statut de review.
- Sous-CRUD MCP après création agent ; aucun ajout au JSON `CreateAgentInput`.
- Port MCP exporté vers `ChatModule`, sans dépendance MCP → chat.
- `ToolRegistryService.prepare()` produit catalogue immuable par message.
- Aucune découverte réseau pendant chat ; snapshot approuvé uniquement.
- Providers OpenAI/Anthropic M8 inchangés.
- Contrats privés MCP séparés de `AgentChatConfig` public.

## Sécurité obligatoire

- Transport Streamable HTTP ; URL HTTPS uniquement en production.
- Protection SSRF avant chaque appel, redirections interdites.
- Credentials structurés, chiffrés AES-256-GCM, write-only.
- Timeout, taille réponse, nombre outils et schémas JSON bornés.
- Secrets, headers et arguments jamais loggés.
- Modification URL/auth/snapshot invalide approbation.
- Statut, activation et sélection revalidés avant chaque `tools/call`.
- Appels réussis et échoués consomment quota commun M8.
