## 2. Structure des routes chat

**Problème initial** : le groupe `(chat)` ne créait pas le segment `/chat` dans l'URL — les pages résolvaient sur `/` et `/<uuid>`.

**Fix** : ajout d'un dossier `chat/` dans le groupe.

```
app/(chat)/
  layout.tsx               → overlay fixed inset-0 z-[100] (bypass Header/Footer)
  chat/
    page.tsx               → /chat  (picker agents)
    [agentId]/
      page.tsx             → /chat/<agentId>  (interface chat dédiée)
```

**`(chat)/layout.tsx`** — overlay fullscreen sans toucher aux autres routes :
```tsx
export default function ChatLayout({ children }) {
  return (
    <div className="fixed inset-0 flex" style={{ zIndex: 100, background: "#faf9f5" }}>
      {children}
    </div>
  );
}
```

## 5. App desktop Pake

Pake (tw93/Pake v3.10.1) wrappant une URL dans un webview Tauri natif (~5.2 MB).

**Fichiers créés** :
- `pake/build.sh` — script flexible : `./build.sh` (localhost), `./build.sh prod` (claake.com)
- `pake/claake.json` — config Pake

**Scripts `package.json` racine** :
```json
"pake-dev":   "node Pake/dist/cli.js http://localhost:3000/chat ...",
"pake-build": "node Pake/dist/cli.js https://claake.com/chat ..."
```

**DMG final** : `frontendWeb/public/Claake.dmg` (servi en statique par Next.js).

**Diagnostic page blanche** : le DMG production pointe sur `https://claake.com/chat` qui n'est pas encore déployé avec les routes corrigées → webview blanc. Solution : build localhost pour tester en attendant le déploiement.

**Pattern nettoyage avant rebuild** :
```bash
rm -f Claake.dmg frontendWeb/public/Claake.dmg
find Pake/src-tauri/target -name "*.dmg" -delete
find /private/tmp -name "*.dmg" -delete
hdiutil info | grep "rw\." | awk '{print $1}' | xargs -I{} hdiutil detach {}
```

## 8. Logo responsive (CSS modules)

**Règle établie** : zéro `style=` inline, tout dans des fichiers `.css`.

**`components/layout/header.module.css`** :
```css
.logoWrapper {
  width: 12%;
  min-width: 4rem;
  max-width: 7rem;
}
.logo {
  width: 100%;
  height: auto;
}
```

**`components/layout/footer.module.css`** :
```css
.logo {
  width: 60%;
  max-width: 6rem;
  height: auto;
}
```

Logo header : `12%` de la largeur du header, contraint entre `4rem` et `7rem`.
Logo footer : `60%` de la colonne brand.

---

## Règles de développement établies

- **Zéro inline style** : tout CSS dans des fichiers `.module.css` ou `globals.css`
- **Zéro `px`** dans les CSS responsives : utiliser `rem`, `em`, `%`, `vw`, `clamp()`
- **Pake rebuild pattern** : toujours nettoyer les DMGs + montages hdiutil avant de relancer
